## 🏗️ 架构优化建议

### 服务层架构

```
┌─────────────────────────────────────────┐
│           Web Layer (Streamlit)          │
└────────────────┬────────────────────────┘
                 │
┌────────────────▼────────────────────────┐
│          Service Layer                   │
│  ┌──────────┬──────────┬──────────────┐  │
│  │DataSvc   │StrategySvc│BacktestSvc  │  │
│  └──────────┴──────────┴──────────────┘  │
└────────────────┬────────────────────────┘
                 │
┌────────────────▼────────────────────────┐
│          Repository Layer                │
│  ┌──────────┬──────────┬──────────────┐  │
│  │StockRepo │StrategyRepo│BacktestRepo│  │
│  └──────────┴──────────┴──────────────┘  │
└────────────────┬────────────────────────┘
                 │
┌────────────────▼────────────────────────┐
│          Data Layer (Parquet)            │
└─────────────────────────────────────────┘
```

### 数据流架构

```
数据下载 → 数据验证 → 数据存储 → 数据预处理
    ↓           ↓           ↓           ↓
Scheduler   Validator   Parquet    TechnicalIndicators
    ↓           ↓           ↓           ↓
定时触发    质量检查    原始数据    预处理数据
```

### 回测流程优化

```
策略配置 → 参数优化 → 回测执行 → 结果分析 → 报告生成
    ↓           ↓           ↓           ↓           ↓
YAML配置    GridSearch  BacktestEngine  Metrics    ReportGenerator
    ↓           ↓           ↓           ↓           ↓
多策略支持   自动调参    确定性回放   风险分析    PDF/HTML
```

## ⚠️ 风险评估

### 技术风险

| 风险项 | 影响 | 概率 | 应对措施 |
|--------|------|------|----------|
| 数据源不稳定 | 高 | 中 | 自动切换数据源 |
| 回测结果不准确 | 高 | 低 | 添加回测验证 |
| 系统性能瓶颈 | 中 | 中 | 异步任务+缓存 |
| 数据质量问题 | 高 | 低 | 数据验证+监控 |

### 业务风险

| 风险项 | 影响 | 概率 | 应对措施 |
|--------|------|------|----------|
| 策略失效 | 高 | 中 | 参数优化+多策略 |
| 市场异常 | 高 | 低 | 风控模块+熔断机制 |
| 数据延迟 | 中 | 中 | 多数据源+监控 |
