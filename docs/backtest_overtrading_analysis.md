# 回测过度交易问题诊断报告

> 生成时间：2026-07-14
> 回测参数：16只候选股票，2025-01-01 → 2026-05-31（~17个月）
> 回测结果：总交易 1122 次，总收益率 17.01%
> 对标：中际旭创、新易盛、天孚通信 2025年以来涨幅超 10 倍

---

## 一、问题确认：交易频率确实严重偏高

| 指标 | 数值 | 判断 |
|------|------|------|
| 总交易次数 | 1122 次 | 🔴 严重偏高 |
| 回测天数 | ~340 个交易日 | |
| 日均交易 | ~3.3 次/天 | 🔴 几乎每天都在买卖 |
| 候选股票数 | 16 只 | |
| 人均交易 | ~70 次/只（35轮次） | 🔴 每只股票平均每两周就买卖一次 |
| 最大持仓 | 10 只 | |

**对比基准**：如果从2025年初买入中际旭创/新易盛/天孚通信并持有不动，收益率是 1000%+，而策略只有 17.01%。

---

## 二、根因分析：卖出逻辑极度贫瘠

### 2.1 当前卖出逻辑全景

```
should_sell() 被调用时，只有两个卖出触发条件：
  ┌─────────────────────────────────────────────┐
  │ 1. 止损: profit_loss ≤ -15%  → 卖！         │
  │ 2. 止盈: profit_loss ≥ +20%  → 卖！         │
  │ 3. _check_sell_signal()      → 永远返回 False │
  └─────────────────────────────────────────────┘
```

**关键发现**：`SelectionStrategy` 和所有 4 个子策略（Trend/Momentum/VolumePrice/Quality）**都没有重写 `_check_sell_signal()`**。这意味着策略只知道"亏了15%就跑"和"赚了20%就跑"，完全不懂得"趋势还在就继续持有"。

### 2.2 死循环示意图

```
在一个 16只股票、max_positions=10 的环境中：

股票 A 上涨 20%
  → 触达 take_profit=+20%
  → 卖出！  ❌（如果持有至今可能涨 500%）
  → 腾出一个仓位
  → 当天评分最高的股票（可能就是刚卖的 A，或另一个热门股）买入
  → fill 满 10 个仓位

股票 B 下跌 15%
  → 触达 stop_loss=-15%
  → 卖出！  ❌
  → 冷却期 5 天后
  → 如果 B 反弹了，评分回归 → 又买入！
  → 如果 B 继续跌 → 刚躲过一劫

高波动股票 C（如光通信板块）
  → 经常单日 ±5%~10% 波动
  → 两周内可能先 +25% → 卖出
  → 一周后回调 -12% → 评分仍高 → 买入
  → 又涨 20% → 卖出
  → 反复循环

结果：每只股票平均被买卖了 35 次（70次交易÷2），
     像割韭菜一样反复进出，每次都截断了涨幅。
```

### 2.3 量化分析

**为什么 1122 次交易这么多？**

$$
\text{日均交易} = \frac{1122}{340} \approx 3.3 \text{ 次/天}
$$

在一个 16 只股票、满仓 10 只的系统中，当天只要有任何一只触发止盈/止损，就会腾出仓位，策略立即用当天评分最高的股票填上。光通信三巨头在2025年极端强势，它们的涨跌波动足以频繁触发 ±20% 和 -15% 的阈值。

**为什么收益率只有 17.01%？**

1. **过早止盈截断了利润**：像中际旭创这种涨 10 倍的股票，策略在 +20% 就卖了。之后虽然可能再买回来，但买回来的价格可能已经比卖出价高出 50%+，再涨 20% 卖出时，总收益严重折损。

2. **频繁交易的摩擦成本**：
   - 每轮买卖（一买一卖）：佣金 0.03% × 2 + 印花税 0.1% = 0.16%
   - 561 轮 × 0.16% = 约 90% 的本金被摩擦成本消耗在反复交易中
   - 加上滑点成本（slippage），实际损耗更大

3. **止损在震荡中被反复触发**：高波动股票短期回撤 -15% 后反弹是常态，策略却在底部卖出、反弹后又高位买回。

4. **没有"让利润奔跑"的机制**：策略的核心假设是所有因子的得分变化意味着"这个股票变差了"，但实际上对于超级强势股，短期得分波动（如 RSI 短暂超买、MACD 短暂死叉）并不改变长期趋势。

## 四、改进方案（已实施）

> 方案确认时间：2026-07-14，经讨论后确认以下 5 点设计决策：
> 1. ✅ 保留小幅固定止盈（80%）作为兜底保护
> 2. ✅ 极端量价背离（严重缩量涨/放量暴跌）可单独触发卖出
> 3. ✅ 趋势恶化卖出与止损卖出使用不同的冷却期
> 4. ✅ 核心仓位（浮盈 Top-3）享受更大波动容忍度
> 5. ✅ MA20/MA60 足够，不需要 MA120/MA250

### 4.1 五维卖出信号架构

卖出信号分为 5 个维度，按优先级从高到低依次检查：

| 优先级 | 维度 | 信号 | 冷却期 |
|--------|------|------|--------|
| 1 | 硬止损 | 浮亏 ≤ -12% | 10 天 |
| 2 | 趋势结构破坏 | MA 死叉 / MA20 拐头 / ADX 衰竭 | 20 天 |
| 3 | 得分恶化 | 得分 < 买入分 × 0.55 / 跌破 50 分 | 20 天 |
| 4 | 极端量价背离 | 严重缩量涨 / 放量暴跌 | 15 天 |
| 5 | 动量+量价联合 | 动量衰竭 AND 量价背离同时满足 | 15 天 |
| 6 | 移动止盈 | 从最高点回撤 > 15%（核心仓位 20%）| 15 天 |
| 7 | 时间止损 | 持有 > 60 天且浮亏 > -5% | 20 天 |
| 8 | 保底止盈 | 浮盈 ≥ 80% | 15 天 |

### 4.2 核心仓位机制

- 持仓中浮盈最高的前 3 只股票自动成为核心仓位
- 核心仓位享受：移动止盈回撤放宽到 -20%（普通 -15%）、RSI 阈值放宽 10 点
- 这样中际旭创等超级牛股在正常波动中不会被误卖

### 4.3 关键改动

| 改动项 | 位置 | 说明 |
|--------|------|------|
| `Position` 新增 `buy_score`、`highest_price` | `base_strategy.py` | 记录买入得分和持仓期最高价 |
| `should_sell()` 完全重写 | `base_strategy.py` | 五维趋势恶化逻辑，返回 `(bool, reason)` |
| 趋势检测子方法 | `base_strategy.py` | `_check_trend_breakdown`、`_check_score_deterioration` 等 |
| `execute()` 适配新格式 | `base_strategy.py` | `sell_signals` 改为 `[(code, reason)]` |
| 冷却期区分 | `base_strategy.py` | `_last_sell_dates` 改为 `{code: (date, reason)}`，按原因查 `_cooldown_map` |
| 回测引擎适配 | `backtest_engine.py` | `_execute_sell` 传递 sell_reason、`_execute_buy` 记录 buy_score |
| 配置节点新增 | `settings.yaml` | `strategy.sell_signals` — 所有阈值可配置 |

## 五、总结

**根本原因只有一句话：策略只会根据固定的盈亏百分比卖出，不会根据趋势变化卖出。**

这导致：
- 超级牛股（涨 10x）在 +20% 就被截断 → 卖飞了
- 普通股在 ±20% 和 -15% 之间反复震荡 → 反复交易吃手续费
- 止损后无充分冷却 → 刚卖就买，买完又跌
- 16 只股票几乎满仓轮动 → 系统性地频繁换手

**最直接的修复**：用"趋势恶化卖出"取代"固定止盈卖出"。让中际旭创这样的股票在涨到 +20% 时不卖（因为趋势还在），一直持有到趋势结束为止。
