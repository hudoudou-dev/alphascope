### 1.2 缺失数据处理机制

所有子策略的因子评分完成后，统一走 `_redistribute_scores()`。当前模式为 `redistribute`（默认）：

| 模式 | 行为 | 当前使用 |
|------|------|----------|
| `redistribute` | 缺失因子剔除，剩余因子权重归一化 | ✅ 当前默认 |
| `neutral` | 缺失因子按50分计入 | |
| `penalize` | 缺失因子按30分计入 | |
| `exclude` | 任一因子缺失→标记insufficient | |

实际影响：基本面因子中 `roe/debt_to_equity` 数据缺失是常态，`redistribute` 模式下 QualityStrategy 中 `fundamental` 的 30% 权重会被重新分配给波动率/偏度/下行风险中的有效因子。

## 第二部分：核心问题诊断 —— 为什么中际旭创/新易盛赚不到钱

### 2.1 问题定位：策略存在系统性"反强趋势"偏向

通过对评分函数的逐因子推演，发现当前策略在三个关键维度上对「强势趋势股」存在系统性惩罚：

#### 🚨 致命问题1：MomentumStrategy 的"倒U型"动量打分

这是最核心的问题。所谓的「动量策略」实际上是一个**"适中动量偏好"策略**，对强动量股票反而扣分：

```
10日动量打分曲线:
   ret ≤ -5%  → 35 (跌多了给低分，合理)
   -5%<ret≤0  → 48
   0<ret≤5%   → 58
   5%<ret≤15% → 65  ← 最高分！
   ret > 15%  → 55  ← 涨太多反而降分！中际旭创常见的10日涨幅

20日动量打分曲线:
   ret ≤ -10% → 30
   -10%<ret≤0 → 45
   0<ret≤5%   → 60
   5%<ret≤15% → 68  ← 最高分！
   15%<ret≤30%→ 62
   ret > 30%  → 50  ← 涨超30%大降分！新易盛常见的20日涨幅

60日动量打分曲线:
   ret ≤ -15% → 28
   -15%<ret≤0 → 42
   0<ret≤10%  → 60
   10%<ret≤30%→ 68  ← 最高分！
   ret > 30%  → 60  ← 再次降分
```

**结论**：MomentumStrategy 的三个动量周期全部呈现「倒U型」打分——涨得适中分最高，涨太多反而降分。这不是动量策略，这是「温和趋势」策略。

#### 🚨 致命问题2：短期反转因子的纯反转逻辑

```
5日涨跌幅 → 得分:
   -15%  → 78 (最高分！鼓励抄底暴跌股)
   -10%  → 72
   -7%   → 65
   -3%   → 58
   0%    → 52
   +5%   → 48
   +10%  → 40
   +20%  → 30
   +25%  → 22 (最低分！热门股5日涨20%+ → 几乎零分)
```

短期反转因子占 MomentumStrategy 的 35% 权重。**热门趋势股连续上涨时，这个因子给出的是系统最低分**。

#### 🚨 致命问题3：RSI 对超买区间的惩罚

```
RSI≥75 → 25 分（最低分）
```

强势趋势股的 RSI 常年在 70-85 区间，RSI 因子持续给出 25 低分。而这占 MomentumStrategy 的 30% 权重。

### 2.2 数值推演：热门趋势股 vs 平庸股对比

以中际旭创在强势上涨阶段的典型数据做推演：

| **TrendStrategy** (30%) | ADX(80)×0.30 + MA(85)×0.30 + MACD(68)×0.20 + 回调(50)×0.20 | **= 73.1** |
| **MomentumStrategy** (25%) | ⚠️ short_reversal(22)×0.35 + multi_mom(55×0.4+50×0.35+60×0.25=54.5)×0.35 + RSI(25)×0.30 | **= 34.4** ⚠️ |
| **VolumePriceStrategy** (25%) | 量比(60)×0.30 + 换手率(70)×0.15 + 量价相关(72)×0.20 + OBV(65)×0.20 + 缩量止跌(48)×0.15 | **= 63.3** |
| **QualityStrategy** (20%) | ⚠️ 波动率(25)×0.30 + 偏度(63)×0.20 + 下行(38)×0.20 + 基本面: PE=60无PE分、PB=8→20-16=4 +4分 → (50+4)/1.1=49.1×0.30 | **= 42.0** ⚠️ |

**融合得分 = 73.1×0.30 + 34.4×0.25 + 63.3×0.25 + 42.0×0.20 = 21.93 + 8.60 + 15.83 + 8.40 = 54.76**

加长期强势+3 → **57.76**（刚好过50阈值，但排名靠后）

#### 场景：某平庸股（温和上涨）

假设：ADX=22、PDI>MDI、部分多头、MACD金叉、不在回调中、5日涨+3%、10日涨+8%、20日涨+12%、60日涨+15%、RSI=48、量比=1.2、换手率=5%、量价中性、OBV>MA5、波动率=35%、PE=20、PB=3

| 子策略 | 因子得分拆解 | 子策略总得分 |
|--------|-------------|-------------|
| **TrendStrategy** (30%) | ADX(65)×0.30 + MA(75)×0.30 + MACD(68)×0.20 + 回调(50)×0.20 | **= 65.6** |
| **MomentumStrategy** (25%) | short_reversal(48)×0.35 + multi_mom(65×0.4+68×0.35+68×0.25=66.8)×0.35 + RSI(50)×0.30 | **= 55.2** |
| **VolumePriceStrategy** (25%) | 量比(50)×0.30 + 换手率(70)×0.15 + 量价相关(50)×0.20 + OBV(65)×0.20 + 缩量止跌(48)×0.15 | **= 55.7** |
| **QualityStrategy** (20%) | 波动率(55)×0.30 + 偏度(50)×0.20 + 下行(52)×0.20 + PE=20→(30-12=18) PB=3→(20-6=14) → (50+18+14)/1.2=68.3 ×0.30 | **= 57.4** |

**融合得分 = 65.6×0.30 + 55.2×0.25 + 55.7×0.25 + 57.4×0.20 = 19.68 + 13.80 + 13.93 + 11.48 = 58.89**

**对比结论：**

| 股票类型 | Trend | Momentum | VolumePrice | Quality | 融合得分 |
|----------|-------|----------|-------------|---------|----------|
| 🔥 中际旭创（强趋势） | 73.1 | **34.4** ⚠️ | 63.3 | **42.0** ⚠️ | **54.76** |
| 📊 某平庸股（温和） | 65.6 | 55.2 | 55.7 | 57.4 | **58.89** |

> **平庸股得分 (58.89) > 强趋势股得分 (54.76)**

这就是回测赚不到钱的根因：**策略会把涨幅最大的股票排在平庸股之后，自然选不到真正的牛股**。

### 2.3 各子策略对不同类型股票的偏向性总结

```
                    强趋势牛股    温和趋势股    震荡股      超跌股      低波价值股
                    (中际旭创)   (普通白马)   (横盘票)   (暴跌票)    (银行股)
TrendStrategy        ✅ 友好      ✅ 友好      ❌ 一般     ❌ 不友好    ❌ 不友好
MomentumStrategy     ❌ 惩罚      ✅ 友好      ✅ 友好     ✅ 最优      ❌ 一般
VolumePriceStrategy  ✅ 友好      ✅ 友好      ✅ 友好     ❌ 一般      ❌ 一般
QualityStrategy      ❌ 惩罚      ✅ 友好      ✅ 友好     ❌ 惩罚      ✅ 最优
────────────────────────────────────────────────────────────────────────────
加权融合(固定权重)   ⚠️ 偏负面    ✅ 最优      ✅ 良好     ⚠️ 中性     ⚠️ 中性
```

**核心矛盾**：当前策略的"最优画像"是「温和趋势 + 适中动量 + 中等波动 + 合理估值」——这恰好是牛市中涨幅最平庸的一类股票。真正的牛股因为「涨太多、波动大、PE高」被系统性地排除在外。

## 第三部分：其他已识别问题

### 3.1 基本面因子形同虚设

- `roe` 和 `debt_to_equity` 在 tushare `daily_basic` 接口中不存在（属于财报数据），实际无法获取
- `pe_ttm` 和 `pb` 虽已接入，但 QualityStrategy 的评分公式 `max(0, 30-PE×0.6)` 要求 PE<50 才有正向贡献
- 中际旭创 PE 高达 60+，只有 PB 贡献约 4 分，最终 `fundamental` 因子得分约 49 分（低于中性50）
- 这意味着基本面维度**对高成长股是负贡献**

### 3.2 横截面标准化和行情自适应默认关闭

- `cross_sectional_enabled = false` → 所有打分都是"绝对分"，在牛市中分数通胀、熊市中分数通缩，失去跨时期可比性
- `regime_enabled = false` → 融合权重永远固定 30/25/25/20，牛市中本应加重 Trend+VolumePrice，但实际权重不变

### 3.3 OBV 因子区分度极低

OBV 只有二元判断（>MA5 → 65, ≤MA5 → 40），信息利用率极低。热门趋势股 OBV 通常强势上升，二值化后只有 65 分，无法体现趋势强度差异。

### 3.4 回调买点因子在趋势股上不友好

`close ≥ ma5`（即不在回调中）→ 统一给 50 中性分。对于强势趋势股，不需要回调也是好事，但这个因子把「不回调节」和「不确定」混为一谈。

## 第四部分：优化建议

### 维度一：策略逻辑优化（高优先级）

#### S1 🔴 修复动量因子的倒U型打分 → 改为单调递增

**当前问题**：`mom10/mom20/mom60` 的评分在涨幅适中处最高，涨幅过大反而降分。

**建议**：将多周期动量改为单调递增或至少S型递增：

```
10日动量（改为单调递增）:
   ret ≤ -5%  → 30
   -5%<ret≤0  → 45
   0<ret≤5%   → 58
   5%<ret≤15% → 70
   ret > 15%  → 82  ← 不再降分，而是给最高分

20日动量:
   ret ≤ -10% → 25
   -10%<ret≤0 → 42
   0<ret≤10%  → 58
   10%<ret≤20%→ 72
   ret > 20%  → 85  ← 不再降分
```

**影响**：这是解决中际旭创问题的**最关键修改**，直接让 MomentumStrategy 不再惩罚强趋势股。

#### S2 🔴 重新设计短期反转因子的角色

**当前问题**：`short_reversal` 是纯反转因子（涨→低分，跌→高分），在趋势行情中持续给出反向信号。

**建议方案A**：降低短期反转在 MomentumStrategy 中的权重（从 35% → 15%），将释放的 20% 加到多周期动量上。
**建议方案B**：引入"趋势中的回调"判断——只有在中期趋势向下的情况下，短期超跌才给高分；中期趋势向上时，短期回调给中性分而非高分。

#### S3 🟡 RSI 阈值调整

**当前问题**：RSI≥75 直接给 25 分，对强趋势股过于严厉。

**建议**：
- RSI≥75 从 25 → 35（适当放宽）
- 或引入 RSI 趋势判断：RSI 高位但仍在上升（动量加速）给中性分；RSI 高位但拐头向下（动量衰竭）才给低分

#### S4 🟡 回调买点因子增加"不回调节也是好事"的场景

**当前问题**：`close ≥ ma5`（不回调节）统一给 50 分。

**建议**：区分场景：
- `close ≥ ma5` + 均线多头排列 → 65（强势拉升中，无需回调）
- `close ≥ ma5` + 均线空头排列 → 45（下跌趋势中的反弹）
- `close < ma5` 保持现有回调评分逻辑

#### S5 🟡 波动率因子引入方向性

**当前问题**：波动率越高分数越低，不区分上涨波动和下跌波动。

**建议**：将波动率分解：
- 上行波动率（正收益日的标准差）：高→趋势强劲
- 下行波动率（负收益日的标准差）：高→风险大（保持现有惩罚）

对上行波动高+下行波动低的股票不再惩罚，甚至给予加分。

#### S6 🟢 增强 OBV 因子的区分度

**当前问题**：仅二元判断（65 vs 40）。

**建议**：引入 OBV 斜率或 OBV 与价格的背离度，增加信息量：
- OBV 持续上升且斜率增大 → 80
- OBV 上升但斜率减小 → 60
- OBV 横盘 → 50
- OBV 下降 → 35

### 维度二：超参优化

#### H1 🔴 调整子策略融合权重

**当前**：Trend(30%) + Momentum(25%) + VolumePrice(25%) + Quality(20%)

**建议**（用于趋势行情）：
- Trend(35%) + Momentum(20%) + VolumePrice(30%) + Quality(15%)
- 降低被 Momentum 拉低的风险，提高 VolumePrice 中量价齐升因子的贡献

**更优方案**：启用 `regime_enabled`，让 BULL 行情下自动切换到 Trend(45%)+Momentum(30%)+VolumePrice(15%)+Quality(10%)，同时配合 S1 修复后的 Momentum 打分。

#### H2 🟡 降低 `min_score_threshold`

**当前**：50.0（刚好是中位数）

**说明**：在修复 S1 后，强趋势股的得分会显著提升，但考虑到 Top-N 排名机制，阈值本身影响不大。保持 50 即可。

#### H3 🟡 基本面评分放开 PE 上限

**当前**：PE>50 不参与基本面评分。

**建议**：将 PE 阈值放宽到 100 或采用分位数评分，而非绝对阈值：
- PE 在行业中位数 0.5~2.0 倍 → 高分
- PE 过高但增速也高 → 中性分
- PE 为负 → 低分

### 维度三：代码精简

#### C1 🟢 删除 `FundamentalIndicators` 中未被调用的方法

以下方法有定义但策略层和 Web 层均未调用：
- `calculate_ps`, `calculate_roa`, `calculate_gross_margin`, `calculate_net_margin`
- `calculate_dividend_yield`, `calculate_earnings_growth`, `calculate_revenue_growth`
- `filter_by_pe`, `filter_by_market_cap`

**建议**：确认无引用后删除，或保留为未来扩展接口（加注释标注）。

#### C2 🟢 清理 `indicators.weights` 死配置

`config/settings.yaml` 中 `indicators.weights` 节点（ma/rsi/macd/volume 权重）是旧版综合评分的残留，当前多策略架构完全不再使用，建议删除。

#### C3 🟢 统一 `selection` 配置中 5 因子权重

`SelectionConfig` 和 `config/settings.yaml` 中同时存在一套 `trend_weight / momentum_weight / volume_weight / volatility_weight / fundamental_weight`（5因子）配置，但这些权重在当前多策略架构中**未被任何子策略使用**。实际生效的是 `sub_strategies` 下的子策略融合权重。建议删除或在注释中标记为 deprecated。

## 第六部分：总结

当前策略的核心矛盾可以归结为一句话：

> **策略被设计用来寻找"温和上涨"的股票，但牛市中真正赚钱的是"强势上涨"的股票。**

四个子策略中，TrendStrategy（30%）对强趋势股友好，但被 MomentumStrategy（25%）和 QualityStrategy（20%）的系统性惩罚所抵消。MomentumStrategy 的三个动量因子全部采用倒U型打分——涨幅适中最优、涨幅过大降分——这与"动量"的基本定义相悖。QualityStrategy 的低波动+低PE偏好进一步排斥了高成长高波动的热门标的。

修复路径非常清晰：**将动量打分改为单调递增（S1）**，配合**启用行情自适应权重（H1）**，就能让策略在牛市中自然地倾斜向真正的强势股。这是最小改动、最大收益的优化方向。
